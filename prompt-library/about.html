About prompt-library

A curated, zero-dependency library of system prompts engineered for production use — not demos, not tutorials.

What this is

prompt-library is a searchable collection of system prompts that have been tested against real models in real workflows. Every entry defines what the prompt does, the temperature it works best at, which models it has been run on, and where it breaks. You search, filter by category or complexity, read the full prompt text, and copy it in one click.

The site is entirely static — no login, no tracking, no backend. It opens as a local file:// and deploys to GitHub Pages. The whole thing is under 5 files.

Why it exists

Most prompt repositories are tutorials optimised for demos. They show you that something works, not how to make it reliable. When you are building something real — an agent, a product feature, an internal workflow — you need prompts that produce structured output, handle edge cases, and fail predictably. That gap is what this library fills.

The target reader is a sceptical senior engineer or analyst, not a newsletter subscriber. Nothing “revolutionary”. No price predictions. No vibes.

What “production-ready” means

  • structured output The prompt defines the shape of the answer. Headers, lists, JSON, tables — something predictable that downstream code or humans can act on.
  • stated limits The notes field names failure modes, not just tips. A good prompt knows what it cannot do.
  • actually tested The models array lists models it has actually been run against. If it is not tested, it is not in here.
  • neutral tone No shill language. Banned: “revolutionary”, “next-gen”, “game-changing”. Analytical register throughout.
  • specific problem Solves one concrete, real problem. A reformatted “write me a blog post” prompt is not useful. It has to do something distinct.

Open source

prompt-library is MIT licensed. Fork it, extend it, use it commercially. If you have a prompt that meets the quality bar, open a pull request — the contributing guide explains the schema and the review process. If you want to add a whole category, add a key to CATEGORIES in prompts.js and ship two prompts with it. The UI absorbs it automatically.
