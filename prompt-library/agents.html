The agent directory

A practical reference for building reliable AI agents — patterns that work in production, principles that prevent common failure modes, and prompts tuned for agentic contexts. No vague architectures; everything here maps to actual system prompt design.

frameworks & models
patterns
ReAct Loop
intermediate

Interleaves reasoning and acting in a tight loop: think, choose a tool, observe the result, repeat. The most widely deployed agentic pattern and a reliable baseline before reaching for anything more complex.

  • Explicit Thought / Action / Observation structure in the prompt
  • Tool list with usage constraints described inline
  • Max-step budget to prevent runaway loops
  • Stop condition: when to declare done vs. escalate

Orchestrator + Workers
advanced

A planner agent decomposes a task into subtasks and dispatches them to specialised subagents. Each worker has a narrow scope. The orchestrator aggregates results and handles failures. Scales to complex, multi-domain workflows.

  • Orchestrator prompt defines decomposition strategy
  • Worker prompts are scoped and stateless where possible
  • Handoff format: structured JSON between agents
  • Failure modes: timeout, partial result, and retry policy

Tool-Use Scaffold
beginner

Gives a single model a curated toolset and a decision policy for when and how to use each one. Simpler than full ReAct — no loop required. Good for retrieval-augmented generation, calculator tasks, and API lookups.

  • Tool descriptions written as capability constraints, not just names
  • One tool per call policy to reduce ambiguity
  • Citation requirement: cite the tool and result used
  • Graceful fallback when a tool returns empty

Self-Critique Loop
intermediate

The model produces a draft, then critiques it against explicit criteria before returning output. Can be a single model in two passes or two separate calls. Reduces obvious errors without human review for well-defined domains.

  • Critique rubric defined in the system prompt
  • Separate critique role from generation role
  • Pass/fail threshold: when to iterate vs. accept
  • Max revision count to bound cost

Memory-Augmented Agent
advanced

Separates working memory (context window), episodic memory (retrieved past interactions), and semantic memory (vector-searched knowledge) into distinct channels. The system prompt defines what gets written, retrieved, and when.

  • Memory write policy: what is worth storing
  • Retrieval trigger: when to query before answering
  • Freshness policy: how to handle stale memories
  • Conflict resolution when memories contradict

Human-in-the-Loop
intermediate

The agent identifies decision points it cannot resolve alone and pauses to request human input. The system prompt defines the escalation taxonomy: which actions need approval, which can proceed autonomously, and how to ask clearly.

  • Autonomy boundaries listed explicitly in the prompt
  • Escalation format: structured question, not open prose
  • State serialisation so the agent can resume cleanly
  • Confidence threshold below which it always asks

design principles
01
Scope before capability

Define what the agent must not do before defining what it can do. Narrow scope reduces failure modes faster than adding guardrails after the fact.

02
Make the stop condition explicit

Agents without a clear done state loop indefinitely. Every agentic prompt needs a success definition and at least one failure exit.

03
Treat tool descriptions as contracts

Write tool descriptions as: what it does, what it needs, what it returns, and when not to use it. Vague descriptions produce misuse.

04
Log reasoning, not just actions

Prompt the agent to externalise its chain of thought before acting. This makes failures debuggable and is usually worth the token cost.

05
Budget tokens like money

Set an explicit step or token budget in the prompt. Agents with no cost awareness will explore exhaustively. Set a limit, then test the edge.

06
Stateless workers, stateful orchestrators

Keep subagent prompts stateless and idempotent. Push state ownership up to the orchestrator. This simplifies retries and parallelisation.

glossary
agent loop
The execution cycle of perceive → reason → act → observe, repeated until a stop condition is met.
tool call
A structured invocation of an external capability (search, code execution, API) made by the model during a reasoning step.
context window
The total token budget available to the model in one inference call. Agentic tasks burn it fast; managing it is a first-class design concern.
grounding
Anchoring model output to retrieved or verified facts, reducing hallucination in long-running tasks where the model can't rely on training alone.
handoff
The structured transfer of state, task context, and partial results from one agent to another in a multi-agent pipeline.
scaffolding
The orchestration code and system prompt structure that wraps a model and gives it access to tools, memory, and control flow.
reflection
A prompted self-evaluation step where the agent critiques its own output against defined criteria before returning a result.
autonomy boundary
An explicit policy line in the system prompt: actions below it are autonomous; above it requires human approval or escalation.
